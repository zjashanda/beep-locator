# 蜂鸣特征定位参考

## 固定模板

- 默认音频：`16000Hz / 单通道 / 16bit signed little-endian PCM`。
- 默认频率：`1000Hz`。
- 默认结构：`约1秒蜂鸣 + 约1秒间隔/静音 + 约1秒蜂鸣`。
- 默认输出：写到待测音频所在目录，文件名为 `{stem}_beep_result.txt`。

## 检测逻辑

1. 将 PCM 或 WAV 解码成整数采样序列。
2. 按 `50ms` 帧计算目标频率分量，默认目标频率为 `1000Hz`。
3. 对每帧计算：
   - 原始 RMS 和 dBFS。
   - 目标频率正弦/余弦分量能量。
   - `tone_ratio = 目标频率分量能量 / 总能量`。
   - 峰值和零样本比例。
4. 用多级阈值扫描目标频率能量，合并连续活动帧。
5. 从活动段里寻找双蜂鸣结构：
   - 单段长度通常 `0.50s ~ 1.55s`。
   - 两段开始时间差通常 `1.70s ~ 2.30s`。
   - 中间间隔通常 `0.45s ~ 1.45s`。
6. 对候选窗口回到原始 PCM 做复核，低于 `tone_ratio < 0.12` 的候选不能作为实际蜂鸣输出。
7. 对重叠候选去重，优先保留 `tone_ratio` 更高、结构更接近 `1s + 1s + 1s` 的候选。

## 置信度

- `tone_ratio >= 0.35`：高可信，频率特征明显。
- `0.20 <= tone_ratio < 0.35`：中等可信，通常可作为实际蜂鸣。
- `0.12 <= tone_ratio < 0.20`：弱可信，需要结合双蜂鸣结构、峰值和上下文。
- `tone_ratio < 0.12`：不要输出为实际蜂鸣。

## 实际蜂鸣与同步推断

必须区分两类结果：

1. 实际蜂鸣：该音频自身在窗口内有非零波形、RMS、峰值和稳定目标频率分量。
2. 同步推断：该音频自身没有蜂鸣波形，但同组同步通道可推断对应时间窗口。

同步推断不能写成实际蜂鸣。若窗口内全零、RMS 为 0、峰值为 0，应输出“未检测到实际蜂鸣特征”，除非用户明确要求另列同步推断窗口。

## 是否需要参考音频

同类蜂鸣已经完成模板沉淀后，不需要每次提供 `你好小奥.wav`。只要待测音频仍使用同样的蜂鸣频率和节奏结构，提供待测音频和格式即可。

需要重新提供参考音频的情况：

- 目标频率不再是约 `1000Hz`。
- 蜂鸣结构不再是 `1秒 + 1秒间隔 + 1秒`。
- 采样率、通道、位深或编码格式不确定。
- 现场录音设备或生成蜂鸣方式发生变化，导致当前阈值明显不适用。

## 时间格式

将秒数转换为 `X分YY秒ZZZ毫秒`：

```text
总毫秒 = round(seconds * 1000)
分钟 = 总毫秒 // 60000
秒 = (总毫秒 % 60000) // 1000
毫秒 = 总毫秒 % 1000
```
